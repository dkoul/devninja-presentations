Let's Talk Money

The Token Problem

What is a Token?

  • "Hello" = 1 token
  • "Tokenization" = 3 tokens
  • 1 token ~ 4 characters in English
  • Every word you send = money
  • Every word AI thinks = MORE money

The Context Window Tax

  • Your system prompt = paid every call
  • Your history = paid every call
  • Tool definitions = paid every call
  • 200K context = 200K tokens billed!
  • It adds up. Fast. Really fast.
"Sending your entire codebase to the AI every message is like Ubering to the bathroom. Technically works, financially questionable."

Savings: Up to 90%

Prompt Caching

Why pay full price for the same system prompt 1000 times?

How It Works

  • First call: Full price (cache write)
  • Next calls: 10% of normal cost!
  • TTL: 5 min (Sonnet), 1 hr (Haiku)
  • Min cacheable: 1024 tokens

Structure Matters!

  • Static content FIRST (system prompt)
  • Tool definitions SECOND
  • Dynamic content LAST (user msg)
  • Exact prefix match required!
A 30,000-token system prompt: $0.09/request normally. With cache hit: $0.009. That's 10x savings on every cached call!
Real Math
Cache Hit Rate > 60% = Winning

Prompt Caching in Practice
09.5 / 18
Simulation

How a User Would Leverage It

Request 1
system_prompt = “You are a coding copilot...”
tools = [search_repo, run_tests]
user_msg = “Explain the auth flow”

Result: cache write
Request 2
same system_prompt + same tool schema
user_msg = “Now explain the deployment path”

Result: cache hit, much cheaper
The user keeps the static context fixed and changes only the latest user message. That makes the prompt prefix reusable and the cache effective.
Practical Pattern
Same prefix, new question

Claude Code Specific

Skills > Context Files

Your CLAUDE.md doesn't need your entire life story.

Bad: Bloated Context

CLAUDE.md = 500+ lines

  • Loaded EVERY conversation
  • Includes rarely used instructions
  • Tokens wasted on context
  • Slow startup, expensive runs
Good: Skills On-Demand

Skills loaded when needed

  • Keep CLAUDE.md under 200 lines
  • Put specialized logic in /skills
  • Invoke with /skill-name
  • Only pay for what you use!

Skills load on-demand only when invoked. Moving 10K tokens of instructions to skills = 10K tokens saved per conversation.

Load Only What You Need

The Hidden Cost

Tame the Thinking

Extended thinking is great, but not for "What's 2+2?"

When to Disable

  • Simple classifications
  • Data transformations
  • Template filling
  • Formatting tasks
  • Quick lookups

The Settings

  • MAX_THINKING_TOKENS=10000
  • Turn off streaming when not needed
  • Use non-thinking models for simple
  • Reasoning traces = output tokens!
"Setting MAX_THINKING_TOKENS is the single biggest lever for reducing spend in Claude Code."

The Sneaky Overhead

Prune Your Tools

That MCP server with 40 tools? You're paying for all of them. Every. Single. Call.

The Problem

  • LLM APIs are stateless
  • Tool schemas sent EVERY request
  • 40 tools = 10-15KB per turn
  • Using 2 tools? Other 38 = waste!

The Fix

  • Audit your MCP servers
  • Remove unused tools
  • Filter tools by relevance
  • Create task-specific configs
Overhead per unused MCP server
10-15KB
Per request. Every request. Forever. Until you fix it.
Unused Tools = Paid Tokens

Strengths

Where AI Crushes It

  • Boilerplate — "Create FastAPI CRUD" → 200 lines in 30s
  • Tests — "Write pytest with edge cases" → 80% coverage
  • Regex — "Match Indian phone numbers" → Saves 30 min
  • SQL — "Convert to window function" → Correct 90%
  • Docs — "Generate docstrings" → Hours in minutes

The Pattern

  • Repetitive, well-documented tasks
  • Clear inputs/outputs
  • Syntax-heavy, pattern-matching
  • Things freshers spent years on

Act II · Failures
17 / 38
Demo: Failures

Where AI Fails Spectacularly

The Hallucination

Ask: "What is leftpad in Python stdlib?" AI confidently describes a function that doesn't exist.

The Logic Fail

"5 apples, eat 2, buy 3, friend gives half of 6." AI often gets this wrong (answer: 9).

The Security Hole

Ask for SQL builder → AI returns string concatenation. Classic SQL injection that compiles.

These demos reliably fail — they build credibility and set up "skills that matter."

Act III · Skills
20 / 38
Skill 1

System Design > Syntax

AI can write code. It cannot:

  • Decide microservices vs monolith
  • Choose Postgres vs MongoDB vs DynamoDB
  • Design for 10M users vs 10K users
  • Make build-vs-buy decisions
  • Navigate organizational constraints

Why This Compounds

  • AI generates code faster → More architectural decisions needed
  • Senior engineers become "AI orchestra conductors"
  • Every project needs someone who sees the whole system

Indian Context

Why System Design Matters Here

FAANG/MAANG Interviews

2-3 rounds are pure system design. Can't fake it with AI in a live interview.

Service Companies

"Solution Architect" > "Senior Developer" pay. Client-facing roles need this skill.

Startups

Small teams = everyone does architecture. Wrong design = costly rewrites.

Resources: Gaurav Sen (YouTube, free), "Designing Data-Intensive Applications" — the bible.

Skill 3

The "AI Sandwich" Technique

  1. Human: Define the problem precisely — context, constraints, examples
  2. AI: Generate draft solution
  3. Human: Evaluate, critique, refine prompt
  4. AI: Iterate based on feedback
  5. Human: Final review, integration, deployment
Bad prompts = bad output. Good prompts = 10x. Great prompts = AI does 80% correctly.

Prompt Engineering

Patterns That Work

Chain of Thought

"Think step by step before answering" — better for complex reasoning.

Few-Shot Examples

Show 2-3 examples of desired output. AI matches the pattern.

Role Assignment

"You are a senior Python developer..." — sets context and constraints.

Structured Output

"Return JSON with these fields..." — predictable, parseable responses.

These patterns have papers behind them. Not magic, just technique.

Act IV · Pitfalls
27 / 38
Pitfall 1

The Dependency Trap

The Risk:

  • Accept AI output without understanding
  • Cargo-cult programming at scale
  • Can't debug when things break
  • Interview failures — can't code without AI

The Antidote

  • Always understand what the code does
  • Explain it back in your own words
  • Practice coding without AI regularly
  • "AI as bike, not wheelchair"

Pitfall 2

Confident and Wrong

Lawyer Case (2023)

Attorneys cited fake cases from ChatGPT, sanctioned by judge.

Source: NY Times

Made-Up Packages

AI suggests pip install x — package doesn't exist, attacker registers it with malware.

Source: Lasso Security 2024

AWS Hallucination

AI recommends boto3.client('s3').upload_optimized() — method doesn't exist.

Rule: If AI suggests a library/method you haven't heard of — verify it exists first.

Pitfall 3

"AI Will Figure It Out"

Things You Still Need to Know:

  • Data structures and algorithms (interview reality)
  • Git, Linux, networking basics
  • Database fundamentals
  • How the internet works
  • Debugging without AI

Why

  • AI explanations require baseline knowledge
  • Interviews still test fundamentals
  • Production debugging is often offline
  • "Prompt better" requires knowing what to ask
